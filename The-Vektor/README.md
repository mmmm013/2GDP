## 4PE — The Universal Process

**4PE (4 Platform Engine)** is a ubiquitous, universally-applicable **process** created in GPEx. It is not a product — it is a process.

Its generic model uses **KKr per industry** (each industry deployment gets its own KKr instance).

### Absolute Components (immutable — always present in any 4PE deployment)

| Component | Definition |
|---|---|
| **STI** | Standard Template Item — a slot/container in the template structure |
| **STI-Slot** | The container space within an STI; houses one or more BTIs |
| **BTI** | Branded Template Item — the content payload that fills an STI-Slot |

These three components are the **absolute, non-negotiable foundation** of every 4PE deployment, every KKr instance, and all of GPEx. They do not change across industries.

### Industry Adaptability

4PE's generic model is designed to adapt to any industry of focus. The STI/STI-Slot/BTI structure stays fixed; the BTI content and KKr configuration varies per industry.
